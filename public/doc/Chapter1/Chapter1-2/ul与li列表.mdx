# HTML `ul` 与 `li`

当一组内容天然就是并列项时，应该使用列表结构，而不是手动在段落前写破折号。  
`ul` 表示无序列表，`li` 表示列表项。

## 一个示例

:::code{html="<section>&#10;  <h1>前端学习清单</h1>&#10;  <ul>&#10;    <li>理解 HTML 语义标签</li>&#10;    <li>掌握 CSS 布局与响应式</li>&#10;    <li>练习 JavaScript 基础交互</li>&#10;  </ul>&#10;</section>" css="body{&#10;  font-family:'Segoe UI',sans-serif;&#10;  padding:24px;&#10;  background:#fefce8;&#10;  color:#422006;&#10;}&#10;section{&#10;  max-width:700px;&#10;  margin:0 auto;&#10;  padding:24px;&#10;  border-radius:18px;&#10;  background:#fff;&#10;}&#10;h1{&#10;  margin:0 0 16px;&#10;  font-size:32px;&#10;}&#10;ul{&#10;  margin:0;&#10;  padding-left:24px;&#10;}&#10;li{&#10;  margin:10px 0;&#10;  line-height:1.7;&#10;}" js=""}
:::

## 为什么要用列表语义

1. 浏览器和辅助技术都能更准确理解结构。
2. 后续想切换成有序列表时更容易调整。
3. 样式和内容结构解耦，不需要手动输入装饰符号。
