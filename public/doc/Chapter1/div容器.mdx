# HTML `div` 元素

`div` 是最常见的通用容器标签，本身没有特殊语义。  
它适合用来组织布局、包裹组件、做分组，但不应该替代本来就有明确语义的标签。

## 一个布局示例

:::code{html="<div class='card-grid'>&#10;  <div class='card'>&#10;    <h2>公告</h2>&#10;    <p>本周五晚上十点开始系统维护。</p>&#10;  </div>&#10;  <div class='card'>&#10;    <h2>更新</h2>&#10;    <p>编辑器新增了示例代码预览能力。</p>&#10;  </div>&#10;</div>" css="body{&#10;  font-family:'Segoe UI',sans-serif;&#10;  padding:24px;&#10;  background:#eef2ff;&#10;}&#10;.card-grid{&#10;  max-width:760px;&#10;  margin:0 auto;&#10;  display:grid;&#10;  grid-template-columns:repeat(2,minmax(0,1fr));&#10;  gap:18px;&#10;}&#10;.card{&#10;  padding:20px;&#10;  border-radius:18px;&#10;  background:#fff;&#10;  box-shadow:0 18px 40px rgba(79,70,229,0.12);&#10;}&#10;h2{&#10;  margin:0 0 10px;&#10;  font-size:24px;&#10;  color:#312e81;&#10;}&#10;p{&#10;  margin:0;&#10;  line-height:1.7;&#10;  color:#475569;&#10;}" js=""}

## 为什么 `div` 常见

1. 它对布局很友好。
2. 配合 class 可以快速建立样式作用域。
3. 在组件化开发里，很多外层包装都需要一个稳定容器。

## 什么时候不该用它

如果内容本身已经是标题、导航、列表、文章、按钮，就应该优先选对应语义标签。  
`div` 是兜底容器，不是默认答案。
