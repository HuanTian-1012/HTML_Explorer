# HTML `p` 元素

`p` 是 HTML 里最基础的文本容器之一，用来表示一段完整的正文内容。  
当一段文字在语义上可以独立成段时，就应该优先考虑使用 `p`。

## 一个最小示例

:::code{html="<article><h1>产品更新说明</h1><p>本次更新修复了导航栏闪动问题，并优化了首屏加载速度。</p><p>如果你正在整理正文内容，每一段独立描述都适合放进 p 元素中。</p></article>" css="body{font-family:'Segoe UI',sans-serif;padding:24px;background:#f8fafc;color:#0f172a;}article{max-width:720px;margin:0 auto;padding:28px;background:#fff;border-radius:18px;box-shadow:0 16px 36px rgba(15,23,42,0.08);}h1{margin:0 0 16px;font-size:34px;}p{margin:0 0 14px;line-height:1.8;color:#475569;}" js=""}
:::

## 什么时候该用 `p`

- 正文说明
- 文章段落
- 需要自然换段的描述文本

## 常见误区

不要把整页所有文本都塞进一个 `p`，也不要在 `p` 里再套块级结构来强行排版。  
如果你表达的是列表、标题、卡片分组，那应该换成更合适的语义标签。

## 总结

`p` 的价值不在“默认有上下间距”，而在“这是一段独立的文本内容”。  
先考虑语义，再考虑样式，会比只追求视觉结果更稳定。
