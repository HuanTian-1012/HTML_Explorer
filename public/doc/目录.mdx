# HTML `h` 元素

HTML 里的 `h1` 到 `h6` 是标题元素，用来表达内容层级。  
它们的职责是告诉浏览器、搜索引擎和辅助技术：“这段内容在整篇页面结构里处于哪一级”。

下面这个例子里，`h1` 是页面主标题，`h2` 是章节标题，`h3` 是章节下的小节标题。

## `h1` 到 `h6` 分别表示什么

- `h1`：当前页面或当前内容区域里最重要的标题
- `h2`：`h1` 下的主要章节
- `h3`：`h2` 下的子章节
- `h4` 到 `h6`：继续向下细分层级

它们不是“六种不同大小的字”，而是六级标题语义。

## 为什么不能只把它当样式标签

很多初学者第一次接触 `h` 元素时，只看到默认字号变大，于是把它当成“加粗放大文本”的快捷方式。这种理解是错的。

标题元素的主要价值有三个：

1. 帮助用户快速扫读页面结构。
2. 帮助屏幕阅读器按标题层级导航内容。
3. 帮助搜索引擎理解页面重点和内容组织方式。

如果你只是想让一段字变大，应该优先使用 CSS，而不是误用标题标签。

## 常见误区

### 1. 只为了字号大而使用标题

错误示例：

```html
<h3>这段字看起来比较醒目</h3>
```

如果它不是一个“三级标题”，那就不应该用 `h3`。

### 2. 标题层级跳级

不推荐这样写：

```html
<h1>页面标题</h1>
<h3>直接跳到三级标题</h3>
```

更合理的写法：

```html
<h1>页面标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
```

### 3. 一个页面出现多个互相竞争的主标题

页面可以有复杂结构，但通常应该先想清楚谁是当前页面最核心的主题，再决定 `h1` 放在哪里。

## 实际使用建议

- 先画内容结构，再选 `h1` 到 `h6`
- 优先保证层级关系正确，再考虑视觉样式
- 不要为了“看起来更大”而滥用标题标签
- 样式交给 CSS，语义交给 HTML

## 总结

`h1` 到 `h6` 的本质是语义化标题层级，不是字号工具。  
写页面时，先问自己“这段内容属于第几层结构”，再决定用哪个 `h` 元素，这样写出来的页面才更清晰、更可维护。
